Introduction

    This application runs on a STM32F4-discovery board and drives a 320x240 pixel VGA screen.
    Different inputs can be given to the application using a UART connection. With the input thats
    given a variety of shapes, text and bitmaps can be displayed on the screen. It's also possible
    to buffer the given inputs and repeat them.

Usasage

    The program runs via a UART connection on pins PA2(Tx) and PA3(Rx) and drives a 320x240 pixel
    VGA screen.It uses the EE-API.lib to drive the VGA screen and the UART connection.

UART connection

    - BaudRate = 115200 baud
    - Word Length = 8 Bits
    - One Stop Bit
    - No parity
    - Hardware flow control disabled (RTS and CTS signals)
    - Receive and transmit enabled
    - Pins PA2(Tx) and PA3(Rx)
    - tested throughput 10 commands/s

possible commands

    line = "lijn,x,y,x2,y2,width,color"
    ellipse = "ellips,x-mp,y-mp,radius-x,radius-y,color,fill"
    rectangle = "rechthoek,x-lo,y-lo,x-rb,y-rb,color,fill"
    traingle = "driehoek,x,y,x2,2,x3,y3,color,fill"
    text = "tekst,x,y,text,fontname,color,style" [style: "normaal","vet","cursief"]
    bitmap = "bitmap,nr,x-lo,y-lo" [nr; 0=happysmily, 1=sadsmiley, 2=arrow-up, 3=arrow-down,
                                4=arrow-left, 5=arrow-right]
    clearscreen = "clearscherm,color"
    wait = "wacht,ms"
    repeat = "repeat"

*The idle line detection of the UART isn't used so all commands should end with an carriage return.

Repeat function.

    To use the repeat function the input commands need to be buffered. To achieve this the program
    needs to get a wait("wacht") command first, the commands after the wait will be buffered. If the
    last command in the buffer is "repeat" everything including the first wait will be repeated.

    NOTE: giving the repeat command while the buffer is empty the program crashes. So make sure you have enough wait time.
